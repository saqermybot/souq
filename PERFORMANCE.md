# Performance & Cost Notes (Firestore)

## قواعد ذهبية لتخفيف الضغط (خصوصاً بسوريا)
1) **Pagination دائماً** (موجود): لا تجلب كل الإعلانات دفعة واحدة.
2) **تقليل حجم الصفحة**:
   - الإعلانات: 12 (حالياً)
   - المفضلات: 30 (تم تخفيضها)
3) **لا تعمل Reads إضافية لكل إعلان**:
   - تجنب قراءة user profile لكل كرت.
   - ضع الحقول الضرورية داخل listing نفسه (sellerName, city, contact...).
4) **الصور**:
   - استخدم صور مضغوطة/مقاسات صغيرة في القائمة (thumbnail).
   - `loading="lazy"` للصور داخل القائمة.
5) **الفلاتر**:
   - حاول تخلي الاستعلامات بسيطة (city + category + type + createdAt).
   - أي فلترة معقدة (مثل مسافة/سنة) خلّيها بالواجهة بعد جلب صفحة نتائج محدودة.

## Checklist قبل كل ميزة جديدة
- هل ستضيف Reads كثيرة؟
- هل يمكن عملها بـ cache محلي (localStorage/sessionStorage)؟
- هل يجب تحميلها فقط عند فتح صفحة معينة؟

## اقتراحات مستقبلية (عند الحاجة)
- فصل بيانات listing إلى:
  - ListingSummary (للقائمة)
  - ListingDetails (تحميل عند فتح التفاصيل)
- تجميع العدادات (views/favorites) داخل listing لتجنب قراءات إضافية.
